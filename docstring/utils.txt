НАЗВАНИЕ:
RAG Utilities — конфигурация и утилиты

SUMMARY:
Конфигурационные переменные из .env, утилиты для логирования, хэширования, работы с путями.
Экспортирует переменные окружения без создания тяжёлых объектов.
Клиенты и модели создаются локально в модулях где используются.

ВЫВОДЫ:
- Только конфигурация: переменные окружения, без создания тяжёлых объектов.
- Утилиты: setup_logging, file_hash, to_posix, clean_text, message_for_log.
- Клиенты создаются локально: ES в build.py/retriever.py, GRAPH_STORE в build.py/tools.py, LLM в chat.py/build.py.
- Модели создаются локально: embed_model в build.py/retriever.py где используется Settings.embed_model.
- load_prompt: утилита для чтения промптов из файлов.

АНТИПАТТЕРНЫ:
- Создавать глобальные клиенты/модели — нарушает инкапсуляцию и замедляет импорт.
- Загружать тяжёлые модели при импорте модуля — модуль становится медленным для всех.

ЗАМЕТКИ:
- Модель эмбеддингов: BAAI/bge-m3, размерность 1024, normalize=True.
- Reranker: BAAI/bge-reranker-large.
- Neo4j: bolt://localhost:7687, credentials из .env.
- Elasticsearch: http://localhost:9200, индексы: rag, manifest.
