НАЗВАНИЕ:
Docker Compose — оркестрация всех сервисов

SUMMARY:
Конфигурация для запуска всех компонентов RAG системы в Docker контейнерах.
Включает Elasticsearch, Neo4j, Kibana, sandbox и их взаимосвязи.

ВЫВОДЫ:
- Elasticsearch: порт 9200, автосоздание индексов через entrypoint.
- Neo4j: порты 7474 (HTTP), 7687 (Bolt), с APOC плагином.
- Kibana: порт 5601, подключение к Elasticsearch для визуализации.
- Sandbox: изолированная среда с knowledge/ монтированным read-only.
- Сети: все сервисы в одной Docker сети для взаимодействия.
- Volumes: персистентное хранение данных ES и Neo4j.

АНТИПАТТЕРНЫ:
- Использовать host networking — нарушает изоляцию контейнеров.
- Не монтировать volumes — потеря данных при перезапуске.
- Пропускать health checks — сложно диагностировать проблемы.

ЗАМЕТКИ:
- Порты: 9200 (ES), 7474/7687 (Neo4j), 5601 (Kibana).
- Volumes: data/elasticsearch, data/neo4j для персистентности.
- Sandbox: knowledge/ монтируется как read-only для безопасности.
- Restart policy: unless-stopped для автоматического восстановления.
