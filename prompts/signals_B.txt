Инструмент: describe_signals_2

Описания полей:
- feature_flags: Фичефлаги. Примеры: feature.flag.x, ab:test:x. Флаги функциональности, A/B тесты, переключатели функций.
- secrets: Секреты вне keys (если выделяете отдельно). Найденные секреты, токены, пароли, ключи API, которые не попали в поле keys.
- permissions: Доменные пермишены. Права доступа на уровне доменной логики, разрешения для операций.
- roles: Роли, RBAC-термины. Роли пользователей, RBAC-роли, группы доступа, уровни привилегий.
- config_keys: Ключи конфигов. Если отделяете от keys, иначе объединить. Ключи конфигурационных файлов, настроек приложения.
- dtos: Имена DTO/схем. Названия Data Transfer Objects, схем данных, моделей для передачи данных.
- entities: Общие сущности домена (если не в symbols). Доменные сущности, бизнес-объекты, если они не включены в symbols.
- domain_objects: Предметные сущности. То же, что entities, лучше объединить с entities. Объекты предметной области.
- bm25_boost_terms: Термы для буста BM25. Сгенерированные ключевые термины для улучшения поиска через BM25 алгоритм.
- io: I/O артефакты (таблицы, топики, файлы, URLs). Общие I/O маркеры, слабый сигнал. Входные и выходные данные, источники и приемники информации.
- tags: Главные ключевые теги, метки. Произвольные метки, категории, теги для классификации и поиска.
- key_points: Ключевые тезисы. Тезисы/summary (для поиска слабые). Основные пункты, ключевые моменты файла.
- likely_queries: Ожидаемые вопросы по файлу. Вероятные запросы (полезно для UX, но не для связности по коду). Вопросы, которые могут задать пользователи по этому файлу.
- security_flags: Флаги безопасности (pii, secrets, crypto, authz, audit). Общие флаги безопасности, если не конкретные ключи/скоупы. Маркеры обработки персональных данных, криптографии, авторизации, аудита.
- todos: TODO/FIXME/XXX. TODO/NOTE/xxx, слабая польза для связности. Заметки о необходимых доработках, комментарии разработчиков.
