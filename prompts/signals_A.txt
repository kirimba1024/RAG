Описания полей:
- symbols: Символы/идентификаторы (универсально). Каноничные имена сущностей, символов, типов, интерфейсов, констант - все значимые идентификаторы, которые используются для связности кода.
- paths: Пути/маршруты/файлы. Примеры: /v1/charges, src/payments/service.ts. HTTP-маршруты, пути к файлам, URL-паттерны, файловые пути в проекте.
- keys: Ключи ENV/CFG/feature/secret. Примеры: ENV.STRIPE_API_KEY, feature.flag.checkout_v2. Ключи конфигурации, переменные окружения, feature flags, секреты в краткой форме.
- api_endpoints: Интерфейсы (REST/gRPC/GraphQL). Примеры: POST /v1/charges, PaymentService/CreateCharge. Конкретные API endpoints с методами, gRPC сервисы, GraphQL операции.
- db_entities: Схемы данных (таблицы/колонки). Примеры: sales.orders, orders.total. Имена таблиц, колонок, представлений, индексов в базе данных.
- dependencies: Внешние зависимости (пакеты/образы/версии). Примеры: npm:axios@1.6, docker:redis:7. Внешние библиотеки, пакеты, Docker образы с указанием версий.
- events_queues: События/очереди (топики/имена). Примеры: event:OrderCreated, kafka:billing.charges. Названия событий, топиков Kafka, очередей сообщений, SNS topics.
- idents: Локальные идентификаторы (нормализованные). Примеры: create_charge, customer_id. Нормализованные имена переменных, функций, констант, используемые для поиска связей в коде.
- headers_auth_scopes: Протоколы и доступ (заголовки/скоупы). Примеры: Authorization, scope:payments:write. HTTP заголовки, OAuth скоупы, роли доступа, токены авторизации.
- errors_codes: Коды/константы ошибок. Примеры: SQLSTATE:23505, ERR_STRIPE_TIMEOUT. Коды ошибок, исключений, статус-коды, константы ошибок.
- imports: Импортируемые модули (индикатор связей по коду). Импорты библиотек, модулей, пакетов - показывает зависимости и связи между файлами.
- functions: Имена функций/методов. Названия функций и методов, если они не включены в symbols или idents.
- classes: Имена классов. Названия классов, интерфейсов, структур, типов данных.
- variables: Имена переменных и констант. Названия переменных, констант, параметров, если они не включены в idents.
