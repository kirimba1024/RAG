Тебе дано только содержимое файла: {rel_path}
Задача — целиком покрыть файл блоками по смыслу: функции/классы, секции конфига, логические группы; если это таблица — дели по строкам.
Границы выбирай так, чтобы по ним было легко находить нужный фрагмент по смыслу без чтения всего файла.
Перекрытия возможны, но нежелательны — избегай без необходимости.
Верни результат одним вызовом split_blocks, заполнив все поля:
• верхний уровень: title (краткий заголовок файла), description (одной фразой), summary (1–3 предложения), language (основной язык/диалект), purpose (зачем файл), file_type (из enum), tags (до нескольких ключевых тегов), key_points (до трёх тезисов);
• blocks: список объектов с start_line, end_line, title, kind.
Ничего, кроме одного вызова инструмента, не выполняй.
